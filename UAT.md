# 安心血壓（SafeBP）

Version：v1 UAT

Status：User Acceptance Test (UAT)

---

# Product Vision

安心血壓不是醫療診斷工具。

目標是：

> 協助長者輕鬆、安心地記錄每日血壓，

> 並提供容易理解的健康資訊，

> 而非造成焦慮。

設計原則：

- 簡單
- 大字體
- 少操作
- 視覺化
- 友善
- 不增加心理壓力

# P1（v1 必做）

## UAT-017 長者輸入與回饋改善

首頁輸入：

- 收縮壓輸入三位數後移至舒張壓。
- 舒張壓輸入 `80`、`99` 等兩位數時，自動移至脈搏。
- 舒張壓第一碼為 `1` 時等待三位數，可輸入 `100`、`120`、`180`。
- 使用者回頭手動修改舒張壓時，不應強制跳欄。

操作回饋：

- 儲存按鈕需同時顯示圖示與「儲存紀錄」。
- 成功、失敗與一般提示需有不同圖示及顏色。
- 提示顯示 5 秒；連續提示不得被前一次計時器提前關閉。
- 錯誤欄位需設定無效狀態，錯誤訊息可由輔助工具朗讀。

保留設計：

- 底部功能列維持純圖示。
- 醫師閱讀內容不全面放大。
- Apps Script 與使用者設定維持管理者操作介面。

## 首頁 Today Card 顯示規則

上次量測：

- 顯示目前使用者最新一筆量測資料
- 不限制必須為今日資料
- 顯示內容包含：
  - 日期時間
  - SYS / DIA
  - Pulse
  - 血壓燈號

日期時間格式：

- 今天：`今天 HH:mm`
- 昨天：`昨天 HH:mm`
- 本週：`週一`~`週日 HH:mm`
- 超過一週：`MM/DD HH:mm`

無資料時：

- 時間顯示 `--`
- 血壓顯示 `-- / --`
- 脈搏顯示 `❤️ -- BPM`
- 狀態顯示 `--`

血壓燈號：

- `🟢 理想`
- `🔵 穩定`
- `🟡 注意`
- `🟠 偏高`
- `🔴 建議觀察`
- `⚫ 建議就醫`

設計原則：

- 120/80 不應被視為偏高
- 優先提供鼓勵、提醒、持續觀察
- 危急值才提供強烈警示

## UAT-011 CSV 匯出

目標：

讓使用者可以把目前紀錄匯出成 CSV，

方便提供給醫師、家人或自行備份。

入口：

- 紀錄頁
- 統計摘要附近提供匯出按鈕
- 按鈕文案：`匯出 CSV`
- 使用下載 / 分享圖示

匯出範圍：

- 預設匯出目前篩選後的紀錄
- 套用快速篩選時：
  - 近7天
  - 近30天
  - 近90天
  - 全部
  - 時段篩選
  - 血壓燈號篩選
  - 自訂日期範圍
- 匯出前顯示目前範圍摘要
- 若未套用篩選，匯出全部紀錄

CSV 欄位：

依照醫師閱讀優先順序：


| 欄位       | 說明                              |
| -------- | ------------------------------- |
| 姓名       | 使用者名稱                           |
| 日期       | `YYYY-MM-DD`                    |
| 時間       | `HH:mm`                         |
| 時段       | 清晨 / 上午 / 下午 / 晚上               |
| SYS      | 收縮壓                             |
| DIA      | 舒張壓                             |
| Pulse    | 脈搏                              |
| 不規則心跳    | 是 / 否                           |
| 血壓燈號     | 理想 / 穩定 / 注意 / 偏高 / 建議觀察 / 建議就醫 |
| RecordID | 紀錄唯一識別碼                         |


規則：

- CSV 第一列必須是欄位名稱
- 日期與時間分開欄位
- RecordID 放最後
- 不修改 Google Sheet Schema
- 不新增後端 API
- 前端使用既有 `getRecords()` 資料產生 CSV
- 數值欄位保持數字
- 文字欄位需處理逗號、換行與雙引號
- 檔案需使用 UTF-8 BOM，避免 Excel 開啟中文亂碼

檔名：

格式：

`SafeBP_血壓紀錄_姓名_YYYYMMDD.csv`

若套用日期範圍：

`SafeBP_血壓紀錄_姓名_YYYYMMDD-YYYYMMDD.csv`

分享支援：

- CSV
- iOS Share Sheet
- AirDrop
- Mail
- LINE
- Files

平台行為：

- 支援 Web Share API 時，優先開啟系統分享
- iPhone / iPad 可透過 Share Sheet 分享到：
  - AirDrop
  - Mail
  - LINE
  - Files
- 不支援 Web Share API 時，改為下載 CSV 檔案
- 桌面瀏覽器使用下載

空資料：

- 沒有紀錄時，不產生空 CSV
- 顯示友善提示：
  - `目前沒有可匯出的紀錄`
- 若篩選後沒有資料，提示：
  - `目前篩選條件沒有可匯出的紀錄`

隱私：

- 匯出前不自動上傳任何資料
- CSV 只在使用者裝置產生
- 不包含 API URL 或內部設定
- 不包含未來同步用的隱藏欄位

驗收標準：

- 在紀錄頁點擊 `匯出 CSV` 可產生 CSV
- CSV 內容與目前畫面篩選結果一致
- Excel / Numbers 開啟中文正常
- iOS Safari 可開啟 Share Sheet
- 不支援分享的瀏覽器可下載檔案
- 無資料時不下載空檔案
- 不影響新增、編輯、刪除紀錄
- 不修改 Google Sheet 欄位

方便：

提供醫師。
