**用途：** UI 設計規範



UI_[GUIDE.md](http://GUIDE.md) 定義整個 App 的設計風格、元件規格與互動方式，確保所有畫面保持一致。



### 設計理念



- Apple Human Interface Guidelines

- Dark Mode 優先

- 長輩友善

- 高對比

- 少步驟

- 大按鈕

- 大字體

### 操作原則



- 點一下即可操作

- 不需要雙擊

- 支援鍵盤 Enter

- 支援觸控裝置

- 優先考慮長輩使用情境

### 首頁輸入與回饋

- 收縮壓輸入三位數後移至舒張壓。
- 舒張壓以 `1` 開頭時允許輸入三位數；其他數字輸入兩位後移至脈搏。
- 使用者回頭手動修改舒張壓時，不強制自動跳欄。
- 儲存按鈕同時顯示圖示與「儲存紀錄」文字。
- 底部功能列因空間限制維持純圖示，並保留輔助工具標籤。
- 成功、失敗與一般提示以不同圖示及顏色區分，顯示 5 秒。
- 輸入錯誤需聚焦欄位，並提供可由輔助工具朗讀的錯誤訊息。
- 醫師閱讀資訊與管理者設定可維持資訊密度，不套用首頁主要操作字級。

### 單一數值紅字

- 收縮壓 ≥ 140 時，只將收縮壓標紅。
- 舒張壓 ≥ 90 時，只將舒張壓標紅。
- 脈搏 < 50 或 > 100 時，只將脈搏標紅。
- 未達紅字門檻的其他數值維持一般文字，不因整筆燈號狀態一起標紅。
- 首頁上次量測、歷史紀錄及趨勢摘要使用相同規則。

### 初次設定教學頁

- 設定頁提供「開啟設定教學」入口，教學在 App 內獨立頁面顯示。
- 教學依序說明建立「血壓紀錄」工作表、貼上 Apps Script、部署網頁應用程式及回填 `/exec` 網址。
- 複製功能直接載入專案根目錄的 `Code.gs`，不得另外維護重複副本，也不在頁面預覽完整程式碼。
- 複製按鈕需複製完整 `Code.gs`；載入或複製失敗時需顯示清楚訊息。
- 教學頁需提供明確的「返回設定」按鈕，底部設定圖示維持選取狀態。

### 設定頁折疊與首次提醒

- 「連線與使用者」、「燈號與紅字規則」及「初次設定教學」預設收合，點擊標題才展開。
- 尚未設定 Apps Script 網址時，啟動後自動前往設定頁並展開「連線與使用者」。
- 首次提醒顯示 5 秒，網址欄位自動取得焦點。
- 未設定網址時不發送首頁與姓名名單 API 請求。

### 顯示模式

- 設定頁提供「Appearance／顯示模式」折疊區域。
- 使用者可選擇淺色或深色模式，選擇後立即套用。
- 顯示模式儲存於 `bp-theme`，下次開啟時沿用。
- 預設使用淺色模式，不依裝置系統主題自動切換。
- 選取按鈕需使用 `aria-pressed` 表示目前模式。

### iPhone 底部安全區

- 底部功能列背景固定貼齊螢幕底部，不使用安全區作為 `bottom` 位移。
- 功能列高度需包含 `env(safe-area-inset-bottom)`，按鈕內容以底部內距避開 Home Indicator。
- 頁面底部保留空間需包含功能列高度與安全區，避免最後內容被遮住。
- 鍵盤開啟時以功能列完整高度向下移出，不重複加計安全區。
